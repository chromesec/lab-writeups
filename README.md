# Offensive Security Lab Writeups

This repository contains sanitized writeups, notes, and walkthroughs from my home offensive security lab. The lab is used to practice penetration testing techniques, validation workflows, exploitation methodology, privilege escalation, post-exploitation analysis, and reporting in a controlled environment.

## Purpose

The goal of this repository is to document hands-on learning and demonstrate practical offensive security skills in a legal lab setting.

Areas of focus include:

- reconnaissance and enumeration
- vulnerability validation
- exploitation workflows
- privilege escalation
- lateral movement concepts
- web application testing
- Active Directory attack paths
- post-exploitation tradecraft
- reporting and documentation

## Lab Environment

This lab is built for controlled testing and research using vulnerable systems and security tooling, including:

- **Kali Linux**
- **Metasploitable**
- **intentionally vulnerable Windows images**
- additional intentionally vulnerable applications, services, and lab targets over time

Typical tooling and workflows may include:

- Nmap
- Metasploit
- Burp Suite
- John the Ripper
- Hydra
- CrackMapExec / NetExec
- Impacket
- Responder
- BloodHound
- Linux and Windows enumeration tools
- manual exploitation and validation techniques

## Repository Structure

### `writeups/`
Detailed walkthroughs and engagement-style notes for individual lab targets, scenarios, or exercises.

Examples:
- Linux target exploitation
- Windows local privilege escalation
- Active Directory attack path walkthroughs
- web app testing notes

### `methodology/`
General testing workflows, checklists, and repeatable offensive security processes.

Examples:
- internal network enumeration workflow
- web application testing checklist
- privilege escalation methodology
- post-exploitation validation steps

### `notes/`
Short-form research notes, commands, observations, and reference material from lab work.

Examples:
- command references
- service enumeration notes
- exploit troubleshooting
- protocol-specific testing tips

### `reports/`
Sanitized pentest-style reports and formal writeups based on lab scenarios.

Examples:
- executive summaries
- technical findings
- risk statements
- remediation recommendations
